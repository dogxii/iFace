## ai-app-001

### Q1 single | 多轮上下文

多轮对话中的上下文窗口、历史裁剪和摘要压缩如何设计？

- [ ] A. 永远保留全部历史原文。

- [ ] C. 只保留用户第一轮消息。
- [ ] D. 历史越长越好，不需要预算管理。

**解释**：多轮上下文的核心是保留有效状态，而不是堆积原文。

### Q2 multiple | 保留内容

多轮对话中通常更应该保留哪些信息？

- [x] A. 当前任务目标

**解释**：保留高价值信息能提高连续性并减少 token 浪费。

### Q3 multiple | 压缩风险

摘要压缩可能带来什么风险？

- [x] A. 事实遗漏
- [x] B. 细节丢失
- [x] C. 语义失真
- [ ] D. 自动提高精度

**解释**：压缩要配合可回放和可验证机制。
