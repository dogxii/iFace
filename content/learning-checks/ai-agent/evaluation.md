# 评测与线上优化测试一下

## eval-001

### Q1 single | 评测定义

AI Agent 或 LLM 应用中的 evaluation 是什么？

- [ ] A. 只看回答是否流畅。
- [x] B. 用可重复、可比较的方法衡量系统在真实任务中的表现。
- [ ] C. 只让开发者随便试几个问题。
- [ ] D. 只统计模型参数量。

**解释**：评测的目标是支撑工程决策，而不是凭感觉判断效果。

### Q2 multiple | 评测价值

为什么评测是系统落地的基础能力？

- [x] A. 能比较不同 Prompt 或模型版本
- [x] B. 能发现回归
- [x] C. 能统一团队判断标准
- [x] D. 能支撑上线门禁
- [ ] E. 能自动替代线上监控

**解释**：评测让 AI 系统从“感觉不错”变成“可比较、可回归”。

### Q3 multiple | Agent 评测对象

Agent 系统评测通常要看哪些对象？

- [x] A. 最终答案
- [x] B. 工具调用链路
- [x] C. 状态和任务完成情况
- [x] D. 成本、延迟和失败恢复
- [ ] E. 只看文采

**解释**：Agent 评测对象往往是整条链路，而不是单次模型输出。
