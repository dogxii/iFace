## rag-001

### Q1 single | RAG 定义

什么是 RAG？它主要解决什么问题？

- [ ] A. 让模型永远不需要上下文。
- [x] B. 先检索外部知识，再让模型基于证据生成答案。
- [ ] C. 只用向量数据库替代 LLM。
- [ ] D. 让模型自动训练知识库。

**解释**：RAG 的核心是 retrieval + generation，用外部证据缓解知识过期、领域缺失和幻觉。

### Q2 multiple | 适用问题

RAG 通常适合解决哪些问题？

- [x] A. 企业知识问答
- [x] B. 文档助手
- [x] C. 客服 FAQ
- [x] D. 内部搜索
- [ ] E. 纯创意写作

**解释**：RAG 更适合需要外部知识支撑的任务，而不是纯发散生成。

### Q3 multiple | 价值边界

关于 RAG 的价值，哪些说法正确？

- [x] A. 能改善知识时效性
- [x] B. 能减少凭空猜测
- [x] C. 能让回答更有依据
- [ ] D. 能彻底消除幻觉
- [ ] E. 能替代所有模型能力

**解释**：RAG 提供证据和上下文，但不是绝对正确性保证。
