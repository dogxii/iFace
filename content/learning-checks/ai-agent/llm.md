# LLM基础测试一下

## llm-001

### Q1 single | LLM 定义

什么更准确地描述了 LLM 相比传统机器学习模型的核心特点？

- [ ] A. 只能做单一分类任务，不能迁移到别的任务。
- [x] B. 先通过大规模预训练获得通用语言能力，再用提示词或微调适配多任务。
- [ ] C. 必须为每个业务单独训练一个小模型才能工作。
- [ ] D. 只能处理自然语言，不能处理代码或结构化信息。

**解释**：LLM 的关键不是“只会回答”，而是先学通用表示，再通过上下文、指令和对齐适配多类任务。

### Q2 multiple | LLM 特征

哪些说法符合 LLM 的工程特征？

- [x] A. 能力通常受模型规模、数据、上下文窗口和对齐方式共同影响。
- [x] B. 可以通过 Prompt、RAG 和工具调用扩展能力边界。
- [x] C. 可能在事实、时效性和精确计算上出错。
- [ ] D. 天然等同于知识数据库，所有事实都可靠。
- [ ] E. 只适合做闲聊，不适合做严肃业务。

**解释**：LLM 是通用底座，不是事实引擎。工程上常用系统能力补足它的不确定性。

### Q3 multiple | LLM 边界

哪些场景通常不必优先上复杂的 LLM 系统？

- [x] A. 规则明确、输入输出固定的简单转换。
- [x] B. 强依赖实时精确数据且可以用确定性接口完成的任务。
- [x] C. 只需要稳定分类或简单抽取且传统方法足够的任务。
- [ ] D. 所有需要高质量自然语言生成的任务。
- [ ] E. 所有带对话界面的产品。

**解释**：是否需要 LLM，先看任务是否需要语言泛化、上下文理解和开放式生成，不要为了“AI 化”而 AI 化。

- [x] A. 上下文截断
